# AMLL TTML Loader 常见问题与排查

当前版本：`1.0.5`

这份文档用于排查 AMLL TTML Loader 使用中的常见问题。完整使用说明见 [USER_GUIDE.md](USER_GUIDE.md)。

## AMLL 歌词没有加载

可能原因：

- 当前歌曲标题、歌手或专辑信息不完整。
- AMLL TTML DB 没有收录这首歌。
- GitHub raw 或相关资源暂时无法访问。
- 自动匹配结果不够可靠，插件回退到了本地歌词。
- 之前匹配失败的记录还在缓存中，插件暂时没有重新请求。

可以尝试：

1. 确认歌曲标题和歌手信息是否正确。
2. 打开插件设置，使用 `手动匹配当前歌曲`。
3. 换一首元数据完整、常见的歌曲测试网络和插件是否正常。
4. 开启运行日志后重试，并查看日志中是否有搜索失败或网络错误。
5. 必要时清理缓存后重新播放。

## 匹配到了错误歌词

可能原因：

- 歌曲同名但版本不同。
- 标题里带有 live、remix、伴奏、翻唱等版本信息。
- 歌手或专辑元数据错误。
- AMLL TTML DB 中的候选结果本身不适合当前音频。

可以尝试：

1. 打开 `手动匹配当前歌曲`。
2. 修改标题、歌手、专辑后重新搜索。
3. 预览候选歌词，选择正确结果后保存。
4. 如果在线结果都不合适，选择使用本地/元数据歌词。

## 手动匹配搜不到结果

可以尝试：

1. 简化歌曲标题，去掉括号里的版本说明。
2. 只保留主要歌手，去掉 featured artist 或额外标记。
3. 检查是否有繁简、日文假名、英文大小写或符号差异。
4. 如果仍然没有结果，可能是 AMLL TTML DB 暂未收录这首歌。

## 歌词时间不准

先判断问题类型：

- 如果整首歌都固定提前或滞后，使用歌词偏移。
- 如果只有某几句不准，通常是歌词文件本身时间轴不适合当前音频版本。
- 如果当前歌曲是 live、remix、剪辑版或平台特殊版本，可能需要换用更匹配的歌词。

调整方式：

1. 播放问题歌曲。
2. 打开插件设置中的歌词偏移调整。
3. 只影响当前歌曲时，设置当前歌曲偏移。
4. 大多数歌曲都同样提前或滞后时，设置全局偏移。
5. 保存后重新播放当前歌曲。

偏移单位是毫秒。正数表示歌词延后显示，负数表示歌词提前显示。

## 本地歌词没有被读取

可以检查：

- 旁挂歌词文件是否和音频文件同名。
- 扩展名是否是 `.ttml`、`.lrc` 或 `.spl`。
- 歌词文件是否放在音频文件旁边。
- 内嵌歌词是否在 FLAC 常见歌词字段中。
- 当前歌曲是否已经被手动固定到某个 AMLL 结果。

如果想强制当前歌曲使用本地歌词，可以打开 `手动匹配当前歌曲`，选择使用本地/元数据歌词。

## 日志在哪里

日志目录：

```text
%APPDATA%\Salt Player for Windows\workshop\amll-ttml-loader-cache\logs
```

插件设置中也提供打开或清理日志目录的入口。

反馈问题时，不要上传完整日志。请只截取问题发生时间附近的相关片段，并遮挡不想公开的歌曲名、路径或其他信息。

## 如何重新匹配当前歌曲

如果当前歌曲已经匹配过，但你想重新选择：

1. 播放这首歌。
2. 打开插件设置。
3. 进入 `手动匹配当前歌曲`。
4. 修改搜索信息或重新搜索。
5. 选择新的 AMLL 结果，或改为使用本地/元数据歌词。
6. 保存后重新播放当前歌曲。

## 如何清理缓存

缓存目录：

```text
%APPDATA%\Salt Player for Windows\workshop\amll-ttml-loader-cache
```

清理方式：

1. 关闭 Salt Player for Windows。
2. 删除上面的缓存目录。
3. 重新启动 Salt Player for Windows。
4. 重新播放歌曲，让插件重新搜索和生成缓存。

清理缓存会删除已转换歌词、匹配记录、手动选择、歌词偏移和日志。清理前请确认这些数据不再需要。
