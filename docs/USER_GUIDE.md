# AMLL TTML Loader 用户手册

本手册面向 Salt Player for Windows 用户，说明如何安装和使用 AMLL TTML Loader。你不需要会写代码，也不需要安装 JDK。

当前版本：`1.0.5`

可以点击页面右侧或 GitHub 页面中的 Outline 按钮查看目录索引，并快速跳转到需要的章节。

## AMLL TTML Loader 是什么

AMLL TTML Loader 是一个 Salt Player for Windows workshop 插件。它会根据当前播放歌曲的标题、歌手、专辑等信息，搜索 AMLL TTML DB，下载并转换逐词 TTML 歌词，让 Salt Player 可以显示更细粒度的歌词效果。

如果在线歌词不可用、匹配结果不可靠、网络超时，或当前歌曲更适合使用本地歌词，插件会回退到本地歌词或 Salt Player 原本的歌词流程。

插件不能保证每首歌都能匹配到准确歌词。匹配效果会受到 AMLL TTML DB 收录情况、GitHub raw 资源访问情况，以及歌曲元数据质量影响。

## 适用对象和使用前提

适合你使用的情况：

- 你正在使用 Salt Player for Windows。
- 你希望优先使用 AMLL TTML DB 的逐词歌词。
- 你的网络可以访问 AMLL TTML DB 和 GitHub raw 相关资源。
- 你的音乐文件尽量有正确的标题、歌手、专辑信息。

普通安装不需要 JDK。JDK 21 只在从源码构建插件时需要。

## 下载与安装方法

1. 打开本项目的 GitHub Releases 页面。
2. 下载当前版本插件包：`AMLL-TTML-Loader-1.0.5.zip`。
3. 将这个 zip 文件复制到 Salt Player for Windows 的插件目录：

   ```text
   %APPDATA%\Salt Player for Windows\workshop\plugins
   ```

4. 重启 Salt Player for Windows。
5. 打开 Salt Player 的插件设置，确认 AMLL TTML Loader 已经出现。

安装时不要解压 zip 文件，直接把 zip 文件放进插件目录即可。

## 第一次使用流程

1. 启动 Salt Player for Windows。
2. 播放一首带有正确歌曲名和歌手信息的音乐。
3. 等待 Salt Player 加载歌词。
4. 如果匹配成功，歌词页会显示 AMLL 来源的歌词。

示例截图：

![自动加载 AMLL 歌词](images/lyrics-page.png)

如果没有加载到 AMLL 歌词，可以先换一首元数据更完整的歌曲测试，或者使用插件设置里的手动匹配功能。

## 自动匹配 AMLL 歌词的工作方式

插件会根据当前歌曲的元数据搜索 AMLL TTML DB，主要使用：

- 歌曲标题
- 歌手
- 专辑

自动匹配时，插件会尽量避免把同名歌曲、不同版本歌曲或元数据错误的歌曲误匹配到一起。搜索失败、等待超时、匹配结果不够可靠时，插件会回退到本地歌词或 Salt Player 默认歌词流程。

自动匹配失败后，插件会记录一段时间，避免同一首歌反复请求失败结果。这个记录会在约 7 天后过期，之后插件会重新尝试匹配。

## 手动匹配当前歌曲

当自动匹配不到歌词，或匹配到了错误歌词时，可以手动匹配。

1. 打开 Salt Player 的插件设置。
2. 点击 `手动匹配当前歌曲`。
3. 在弹窗中检查当前歌曲的标题、歌手、专辑。
4. 如果元数据不准确，可以直接修改后重新搜索。
5. 在搜索结果中选择更合适的 AMLL 歌词。
6. 预览确认后保存。
7. 重新播放当前歌曲，查看歌词是否正确。

手动匹配弹窗示例：

![手动匹配当前歌曲](images/manual-match-dialog.png)

手动选择会为当前歌曲保存一条记录。之后播放同一首歌时，插件会优先使用你手动固定的结果。

## 使用本地/元数据歌词

如果你不想让某首歌使用 AMLL 在线歌词，可以在手动匹配弹窗中选择使用本地/元数据歌词。

适合这样做的情况：

- AMLL 搜索结果不准确。
- 你已经有校对好的本地歌词。
- 这首歌是特殊版本、现场版、翻唱版，在线结果很容易错。

插件支持常见的旁挂歌词文件，例如同名 `.ttml`、`.lrc`、`.spl` 文件，也会尝试读取 FLAC 元数据中的歌词字段。能否读取成功取决于文件格式和播放器可提供的信息。

## 歌词偏移调整

如果歌词整体提前或滞后，可以在插件设置里调整歌词偏移。

插件支持两类偏移：

- 全局偏移：适用于所有没有单独设置偏移的歌曲。
- 当前歌曲偏移：只对当前正在播放的歌曲生效。

操作步骤：

1. 播放需要调整的歌曲。
2. 打开 Salt Player 的插件设置。
3. 点击歌词偏移相关设置。
4. 设置全局偏移或当前歌曲偏移。
5. 保存后重新播放当前歌曲，让新偏移生效。

偏移单位是毫秒：

- 正数：歌词延后显示。
- 负数：歌词提前显示。
- `0`：不额外偏移。

歌词偏移弹窗示例：

![歌词偏移调整](images/lyric-offset-per-track-dialog.png)

如果某首歌设置了当前歌曲偏移，它会覆盖全局偏移。删除当前歌曲偏移后，才会重新使用全局偏移。

## 运行日志的开启、查看和清理

插件设置中可以开启或关闭运行日志，也可以切换普通日志和详细日志。

日志通常用于排查问题，例如：

- 为什么 AMLL 歌词没有加载。
- 搜索到了哪些候选结果。
- 为什么回退到了本地歌词。
- 读取本地歌词或缓存时是否失败。

日志保存在：

```text
%APPDATA%\Salt Player for Windows\workshop\amll-ttml-loader-cache\logs
```

插件会自动清理旧日志，通常只保留最近一段时间或最近若干个日志文件。你也可以在插件设置中打开或清理日志目录。

反馈问题时，请不要上传完整日志。完整日志可能包含歌曲名、歌手、文件路径等个人信息。建议只截取和问题发生时间相关的几行日志，并遮挡不想公开的信息。

## 常见问题

更详细的排查步骤见 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)。

### AMLL 歌词没有加载

可能是歌曲元数据不完整、AMLL TTML DB 没有收录、GitHub raw 资源无法访问，或之前的失败记录还在缓存中。可以先尝试手动匹配当前歌曲。

### 匹配到了错误歌词

打开 `手动匹配当前歌曲`，修改标题、歌手或专辑后重新搜索。找到正确结果后保存。特殊版本歌曲也可以固定使用本地/元数据歌词。

### 手动匹配搜不到结果

AMLL TTML DB 可能没有收录这首歌，或者搜索词不适合。可以简化标题，去掉括号里的版本信息，或只保留主要歌手再试。

### 歌词时间不准

先确认是不是整首歌都固定提前或滞后。如果是，使用歌词偏移调整。只有当前歌曲不准时，优先设置当前歌曲偏移。

### 本地歌词没有被读取

确认歌词文件和音频文件是否同名，扩展名是否是插件支持的 `.ttml`、`.lrc`、`.spl`。如果是内嵌歌词，目前主要支持 FLAC 常见歌词字段。

## 卸载插件与清理缓存

卸载插件：

1. 关闭 Salt Player for Windows。
2. 打开插件目录：

   ```text
   %APPDATA%\Salt Player for Windows\workshop\plugins
   ```

3. 删除 `AMLL-TTML-Loader-1.0.5.zip`。
4. 重新启动 Salt Player for Windows。

清理缓存和设置：

1. 关闭 Salt Player for Windows。
2. 删除缓存目录：

   ```text
   %APPDATA%\Salt Player for Windows\workshop\amll-ttml-loader-cache
   ```

清理缓存会删除已转换歌词、匹配记录、手动选择、歌词偏移和日志。下次使用时，插件会重新生成需要的数据。

## 隐私说明

插件会使用当前歌曲的标题、歌手、专辑等信息请求 AMLL TTML DB，用于搜索匹配歌词。插件不会上传音频文件，也不会上传 Salt Player 账号信息。

本地歌词、缓存、手动匹配记录、偏移设置和日志都保存在本机。日志中可能出现歌曲信息、搜索关键词或本地文件路径。公开反馈问题前，请只提供必要片段，并自行遮挡隐私内容。

如果你不希望插件发起网络请求，可以停用插件，或为具体歌曲固定使用本地/元数据歌词。
