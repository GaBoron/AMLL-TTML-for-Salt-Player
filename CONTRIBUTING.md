# Contributing / 贡献指南

Contributions are welcome. Please keep changes focused and easy to review.

欢迎贡献。请尽量保持修改聚焦，方便 review。

Basic workflow / 基本流程:

1. Fork this repository / Fork 本仓库。
2. Create a branch for your change / 创建修改分支。
3. Modify code or documentation / 修改代码或文档。
4. Build locally:

   ```powershell
   .\gradlew.bat --no-daemon plugin
   ```

5. Submit a Pull Request / 提交 Pull Request。
6. In the PR description, explain the purpose of the change and how it was tested / 请在 PR 中说明修改目的和测试情况。

Please avoid unrelated refactors in bug fixes or small documentation updates.

修复 bug 或小幅文档更新时，请避免混入无关重构。
